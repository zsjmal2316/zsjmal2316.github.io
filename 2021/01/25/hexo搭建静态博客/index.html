Hexo使用melody主题搭建博客

Hexo介绍

Hexo是一个快速、简洁且高效的博客框架，可以通过Markdown解析语法，生成对应的html。安装Hexo只需要几分钟时间，如果只是单单写自己的文章，很推荐使用Hexo

Hexo搭建很简易，只需要几个步骤，但后续添加的功能（如：评论设置、搜索设置以及搬迁到自己的个人域名等…）还是需要花一些时间去操作。可以说是搭起简易，功能难易

推荐使用melody，它是简洁美观的主题，而且支持很多功能，只需要跟着文档套用即可。当然Hexo也有好多主题，不同的主题配置有些不一样，但都是大同小异

hexo-theme-melody主题下载：https://github.com/Molunerfinn/hexo-theme-melody

hexo-thmem-melody主题文档：https://molunerfinn.com/hexo-theme-melody-doc/theme-pages.html

下载 git和node

Hexo是基于与node.js和git来搭建，所以需要安装这些应用程序（下述基于Linux搭建，Windows和Mac都差不多，可以到官网查看安装步骤）

下载git

Linux会默认安装node.js，但版本比较低，不能于hexo兼容，需要移除。所以要去官网下载最新

下载官方地址：https://nodejs.org/zh-cn/download/

做命令的软连接，使其可以在全局上使用

设置淘宝的cnpm加速源，通过cnpm下载hexo会更加快速

安装hexo

安装hexo并初始化目录

初始化目录下的文件如下，文件下的内容可以参考中文官网的解释Setup

_config.yml：网站的配置信息，大部分配置都会在这个文件

package.json：应用程序信息，可以移除自己安装的一些插件

scaffolads：模板目录，可通过模板改变你每次生成的文章内容

source：用户资源存放的地址，如（文档、分类、标签）

themes：主题目录，根据不同的主题来生成静态页面

生成一个页面，并发布到本地

浏览器输入http:localhost:4000，搭建成功

我使用的是购买服务器地址来访问，不是在本地localhost，但操作都是一样的

注意：记得在安全组放通4000端口即可

hexo的命令说明

这里只列出了一些常用的指令，如果想了解更多指令，可到中文官网查看Commands

参数 作用
hexo init [folder] 初始化一个网站目录，如果没有指明目录，则在当前目录下生成
hexo new [layout] 新建一篇文章
hexo g或generate 生成静态文件
hexo s或server 启动服务，通过浏览器访问地址
hexo d或deploy 部署网站，可以部署到GitHub或Coding上
hexo clean 清除缓存文件 (db.json) 和已生成的静态文件 (public)
hexo publish [layout] 发表草稿
hexo list 列出网站资料
hexo version 显示hexo版本

使用melody主题

下载melody主题和渲染插件

主题有两个hexo版本，对应你的版本进行下载和生成对应的文件

如果是跟着我操作部署的，那么版本应该是4.2.0版本

按照文档的配置说明，Hexo低于5.0的，推荐将./node_modules/hexo-theme-melody/_config.yml复制到hexo工作目录下source/_data/melody.yml，如果不存在source/_data目录，则自己创建一个

主题页面和主题功能配置

melody主题设置标签页、分类页、幻听片Slides页面、相册、404页面配置，可以通过作者主题文档来一步步操作，添加自己需要的页面，过程很简单只需要跟着操作就行

主题功能配置有语言、社交图标、导航菜单、文章版权、友链、鼠标点击特效等，需要自己花时间去操作添加，因为每个人的需求都不一样，所以这里就不再操作演示

部署到GitHub

点击右上角的+，选择New repository（创建一个仓库）。仓库名称为：用户名+github.io，这样github才知道你是要部署静态页面

编辑初始目录下的_config.yml文件

type：部署的类型为git

repo：仓库地址

branch：分支默认为master即可

通过命令部署上github

  1. 如果出现部署错误，可能是没有定义用户邮箱和用户名（邮箱不是随意的，而是绑定你的github，可以充当账号登录的邮箱）

    • git config –global user.email “用户邮箱”

    • git config –global user.name “用户名”

  2. 如果已经设置，此时会让你输入账号和密码，输入之后即部署上你的仓库中

查看自己GitHub的仓库，可以看到已经部署上去了

通过SSH免密钥部署Github

上述将自己的博客内容部署到Github，但是每次上传都要需要账号和密码。如果频繁的提交，自己都会觉得不耐烦。所以通过SSH免密要进行部署可以简便，就不需要每次部署都输入账号和密码

生成非对称密钥rsa

查看密钥生成的公钥

点击自己的仓库，进入之后选择settings–>Deploy keys（Add deploy key），然后复制公钥到仓库中

Title：可以自己随意起

Allow write access：需要勾选

编辑初始目录下的_config.yml文件

PS：后续的部署就不需要再输入用户名和密码了

个人域名托管至GitHub

个人域名可以托管到GitHub中，然后就可以通过自己的购买的域名进行访问。当然不使用自己的域名，用GitHub上的（用户名.github.io）访问自己的博客。但如果有自己域名的还是推荐使用自己的域名，方便记忆

我的域名是在腾讯云买的，所以下述的域名解析是腾讯云的，当然其他平台购买的域名也都是这么操作

域名解析，通过CNAME记录将github默认的域名指向自己的域名

注意：添加解析之后，需要一段时间才能访问自己的网站，可能要4-48个小时或更久，如果你网页显示的是503错误（表示：服务器当前忙，暂时无法响应服务器），代表是服务解析的还没成功，需要等待！

在初始目录下source目录中添加CNAME文件

CNAME：内容是自己的域名

清除缓存，重新部署上github

查看是否部署成功，点击仓库–>settings。如果显示Your site is published at xxx就代表部署成功。下拉还需要勾选Enforce HTTPS

评论系统Gitalk

melody主题（第三方）支持很多评论系统，我选择的是Gitalk系统，也是部署在GitHub上，通过github Oauth应用的client id和secret值进行连接.

进入个人设置，选择Developer settings–>OAuth Apps–>New OAuth APP

Application name：可以自己随意起名

Homepage URL：填写github默认的域名

Application description：应用描述，可以自己随意

Authorization callback URL：回调的网址，就写自己的个人域名

在source/_data/melody.yml下开启Gitalk

浏览自己的网站进行测试，登录成功之后，没有报错则成功部署Gitalk

PS：如果出现Not found或其他问题，请确认自己域名设置和仓库名以及用户名有没有填写错误

搜索功能Algolia与Local search

melody主题提供了两种搜索功能，一个是Algolia，一个是local search。这里推荐使用local search，一是方便且也快速，二是相对简单。Algolia虽也快，但毕竟是国外的，搜索过程中需要初始化一些时间。

algolia设置

algolia是一家为网站与移动应用提供托管式搜索API的初创企业，网站或移动应用只需嵌入简单代码数分钟就可以实现搜索功能

algolia有两种安装，即hexo-algolia和hexo-algoliasearch，我使用的是后者，其实两者都相差不多

登录https://www.algolia.com/users/sign_in，可以通过github和Google账号登录

登录后，创建一个index，可以随意起名

选择侧边栏的API Keys

选择All API Keys，然后新建一个API Key

安装hexo-algoliasearch

在初始目录下的_config.yml添加hexo-algoliasearch插件

在初始目录下的_config.yml添加algolia配置

在你的Your API Keys中，都有以下的参数，其余配置说明可以到hexo-algoliasearch查看

在source/_data/melody.yml下开启algolia

部署上algolia，需要将文件的索引上传到algolia，这样才能搜索的到文章标题

如果出现报错，则添加HEXO_ALGOLIA_INDEXING_KEY=API Key（你创建的Key）

浏览algolia，查看是否成功
